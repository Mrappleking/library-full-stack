| API文档 | SpringDoc | 2.3+ | OpenAPI标准，自动生成 |

## 核心设计原则

1. **四层架构**：Controller → Service → Mapper → Database，职责清晰
2. **事务管理**：所有多DAO写入操作必须使用 `@Transactional`
3. **统一响应**：所有API返回 `ApiResponse<T>` 格式
4. **异常处理**：统一异常处理，中文错误消息
5. **缓存策略**：热点数据（规则、分类、图书详情）使用Redis缓存
6. **审计日志**：独立事务记录操作日志

## 关键设计决策

| 决策 | 原因 |
|------|------|
| MyBatis over JPA | 课程教学要求，SQL可控性高 |
| 自定义JwtAuthFilter | 减少依赖，灵活控制认证逻辑 |
| Naive UI over Element Plus | 与原始版本保持一致，深色模式体验好 |
| Axios over fetch() | 课程教学要求，拦截器机制完善 |
| Redis缓存 | 减少数据库压力，提升响应速度 |
| 独立审计事务 | 主业务失败不影响审计记录 |
